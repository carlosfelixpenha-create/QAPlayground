Requisitos 'Tela de Login'

Campos

Usuário

  • Requisito: Deve permitir digitar somente texto (mínimo 3 caracteres). Deve conter nome e sobrenome.
  • Critérios de aceite:
    • Campo não pode estar vazio.
    • Campo não permite dois espaços seguidos.
    • Deve aceitar apenas letras (sem números ou caracteres especiais).
    • Exibir mensagem de erro se não atender às regras.
    • Mensagem exibida: “Preencher corretamente o campo Usuário, dúvida entrar em requisitos!”

Senha

  • Requisito: Deve permitir digitar senha (mínimo 6 caracteres).
  • Critérios de aceite:
    • Campo não pode estar vazio.
    • A senha precisa ter no mínimo um número.
    • A senha precisa ter no mínimo uma letra maiúscula.
    • A senha precisa ter no mínimo um caractere especial.
    • Deve ocultar os caracteres digitados (tipo password), quando fechado.
    • Deve mostrar os caracteres digitados (tipo password), quando aberto.
    • Exibir mensagem de erro se não atender às regras.
    • Mensagem exibida: “Preencher corretamente o campo Senha, dúvida entrar em requisitos!”

Botões

Entrar

  • Requisito: Executa a validação dos campos e simula login.
  • Critérios de aceite:
    • Se usuário e senha estiverem corretos (ex: admin / 1234), exibir mensagem de sucesso.
    • Mensagem exibida: “Login realizado com sucesso!”.
    • Não deve permitir login com campos vazios.
    • Se inválidos, exibir mensagem de erro clara, por campo.

Voltar

  • Requisito: Redireciona para a tela Home.
  • Critério de aceite: Ao clicar, deve carregar home.html.

Requisitos

  • Requisito: Redireciona para a tela de requisitos.
  • Critério de aceite: Ao clicar, deve carregar login-requisitos.html.
